# AGENT SYSTEM DIRECTIVE: MVP-Scoped Future Plans

**CONTEXT**: 
This document outlines lightweight, high-value feature additions that align strictly with a solo-developer MVP scale. Do NOT over-engineer these into massive enterprise subsystems. Keep implementations lean, utilizing existing Supabase rows and React state where possible.

---

## MVP PHASE 1: Historical Analysis Dashboard
**TARGETS**: `frontend/src/pages/DashboardPage.tsx`, `backend/analysis.py`
- **Goal**: Prevent data loss after a session ends.
- **Effort**: Medium | **Time**: ~2-3 AI Prompts | **Cost**: Free (Supabase Tier) | **Dependencies**: None | **Requirements**: Postgres `history` table migration.
- **Scope**: Instead of analysis results vanishing if the user refreshes, build a simple "My Past Analyses" table. Fetch the user's historical `upload_id` runs from Supabase, allowing them to click and re-view past generated themes or download PDFs repeatedly.

## MVP PHASE 2: Lightweight Custom Filtering
**TARGETS**: `frontend/src/pages/UploadPage.tsx`, `backend/services/ai_service.py`
- **Goal**: Give the user more granular control over the AI extraction.
- **Effort**: Low | **Time**: ~1 AI Prompt | **Cost**: Zero | **Dependencies**: None | **Requirements**: React text-input state piped to backend payload.
- **Scope**: Add an optional "Ignored Words" text input field (e.g., "pricing, bug, login") before analyzing a CSV. Pass this string payload to the backend so the LLM is explicitly instructed to exclude specific topics from its thematic clustering.

## MVP PHASE 3: Extraction Confidence Feedback
**TARGETS**: `frontend/src/pages/ResultsPage.tsx`, `supabase_schema.sql`
- **Goal**: Improve your internal prompt engineering over time.
- **Effort**: Low | **Time**: ~1 AI Prompt | **Cost**: Zero | **Dependencies**: Phase 1 | **Requirements**: Supabase schema column update (`is_accurate: boolean`).
- **Scope**: Add simple "Helpful / Not Helpful" feedback buttons next to generated themes. Write a simple boolean (`is_accurate: true/false`) to Supabase so you, as the developer, can see which themes generated by Gemini/ChatGPT are failing.

## MVP PHASE 4: Custom AI Persona Selection
**TARGETS**: `frontend/src/pages/UploadPage.tsx`, `backend/services/llm_service.py`
- **Goal**: Provide varying analysis scopes trivially with zero backend database overhead.
- **Effort**: Low | **Time**: ~1-2 AI Prompts | **Cost**: Zero | **Dependencies**: None | **Requirements**: Enum system prompt arrays in `llm_service.py`.
- **Scope**: Add a simple `<Select>` dropdown allowing the user to choose the "Lens" the AI uses: e.g., *Strict Agile Validation*, *Blue-Sky Ideation*, or *Risk & Churn Analysis*. The backend dynamically swaps the system prompt injected into the LLM based on this single string.

## MVP PHASE 5: Magic Share Links (Read-Only)
**TARGETS**: `frontend/src/pages/SharedResultsPage.tsx`, `backend/analysis.py`
- **Goal**: Allow users to share analysis with their boss/team without building complex multi-tenant architecture.
- **Effort**: Medium | **Time**: ~2-3 AI Prompts | **Cost**: Zero | **Dependencies**: Phase 1 (Requires saved queries) | **Requirements**: Supabase RLS policy adjustment for anonymous external reads via UUID.
- **Scope**: Generate a secure, unique hash URL (`/shared/xyz123`) bound to a specific `upload_id`. Ensure the backend endpoint allows unauthenticated GET requests specifically for that UUID so external clients can view the read-only thematic grid.

## MVP PHASE 6: Sentiment Heatmap (UI Only)
**TARGETS**: `backend/services/llm_service.py`, `frontend/src/components/ThemeCard.tsx`
- **Goal**: Visually highlight urgency in the extracted themes.
- **Effort**: Low | **Time**: ~1-2 AI Prompts | **Cost**: Negligible Token Overhead | **Dependencies**: None | **Requirements**: Update Python Pydantic schemas to validate JSON `sentiment` tags.
- **Scope**: Instruct the LLM to output a `sentiment` field (`positive`, `neutral`, `critical`) for each theme in its JSON response. On the frontend, cleanly map those strings to Tailwind badge UI colors (Green/Gray/Red) without needing to alter any database architecture.

## MVP PHASE 7: Simple A/B Analysis Comparison
**TARGETS**: `frontend/src/pages/ComparePage.tsx`, `backend/analysis.py`
- **Goal**: Allow tracking of customer feedback trends over time.
- **Effort**: Medium | **Time**: ~3-4 AI Prompts | **Cost**: 2x LLM Token Call | **Dependencies**: Phase 1 (Requires saved queries) | **Requirements**: UI Multi-Select and recursive JSON array API handling.
- **Scope**: A simple interface letting the user select two past analyses. The backend sends both previously-generated JSON theme arrays to the LLM and asks for a 2-paragraph synthesis evaluating "What sentiment changed between Dataset A and Dataset B."
